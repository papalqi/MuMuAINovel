# 大纲审查清单（逐条必过）

用法：在你调用 `outline_generate_stream` 得到大纲后，先 `outlines_list` 拉全量，然后按 `order_index` 逐条审查。发现问题就立刻修补，不要等写到几十章再返工。

## A. 全书基调与身份定位（先定“你是谁”）

1. 主角身份属于哪一类：
   - 重生（先知信息差）
   - 穿越（现代知识/思维带入）
   - 穿书（先知 + 改命/救人）
   - 原住民无金手指（贵女/世家训练体系的“系统优势”）
2. 爽点是否围绕身份优势展开：
   - 重生：提前布局、将计就计、扮猪吃虎
   - 穿越：现代知识降维、精神状态美好、不按套路
   - 穿书：改命线强、救亲友、改关键节点
   - 原住民：运筹帷幄、阶层/资源优势压制（但要写得有逻辑）

## B. 结构完整性（每条大纲都要有因果闭环）

对每个大纲节点，至少回答：

1. **目标**：这一段主角要达成什么？
2. **障碍**：阻力来自谁/什么机制？
3. **行动**：主角采取了什么具体动作？
4. **意外**：出现了什么反转/新信息？
5. **结果**：结果改变了什么格局？是否引出下一节点？

> 如果某条只是在“叙述背景/日常”，没有明确冲突推进，就要补冲突或合并/删减。

## C. 五大轻毒点（出现就立刻修）

1. 成长太快：突然变强/变聪明，缺少过渡与代价。
2. 报应太慢：反派恶行出现后，读者期待的“爽点兑现”拖太久。
3. 人设偏离：前期高智，后期恋爱脑；或动机前后矛盾。
4. 单剧情太漫长：一个打脸/误会拉几十章。
5. 换地图太频繁：新配角过密、情感链接断裂。

## D. 五大重毒点（直接弃文级）

- 太弱智、太圣母、总被虐、反三观、拆 CP

## E. 五过度（写的时候最常失手）

1. 过度调皮 → 变蠢
2. 过度深情 → 变油
3. 过度高冷 → 变傲慢
4. 过度深刻 → 变说教
5. 过度善良 → 变圣母

## F. 人物同步（角色表与大纲要对齐）

1. 名称一致：全书统一“姓名/字/外号/尊称”，避免同一人出现多个写法。
2. 动机一致：角色每次关键选择都有动机链（利益/情感/恐惧/欲望）。
3. 能力一致：智商/战力/资源强弱要可解释；要有“成长曲线”。
4. 关系一致：亲属/婚约/组织隶属不乱改；改动必须同步到 `world_rules`。

## G. 地点同步（地图与转场要有牵引）

1. 地名统一：同一地点不要多版本（除非刻意写“别称/方言”，也要统一标注）。
2. 转场理由：每次换地图都要有“主线牵引”（任务/追查/逃亡/升学/调任）。
3. 新角色引入：用“旧角色关系/旧线索”牵引读者接受新配角。

## H. 时间线同步（最容易翻车）

1. 年龄/年份/季节/行程可推导；不要出现“同一天跨城三次”。
2. 若出现冲突：以 `world_rules` 与已发布章节为准，倒推修正大纲。

## I. 审查结论输出（写入可执行修改清单）

对每条大纲，输出一个最小可执行的修补项：
- “补动机一句”
- “把地点统一为 X”
- “把反派报应提前到第 N 条/第 N 章”
- “合并两条大纲，避免拖沓”

然后用 `outline_generate_stream` 逐条修补（不要一次让模型推倒重写全书）。
